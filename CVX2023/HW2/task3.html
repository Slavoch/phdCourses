def get_U_V(Y, T, alpha, U1=None, U2=None):
    if U1 is None:
        D = U2 + T
    elif U2 is None:
        D = U1 - T
    R = D.shape[1]
    mat = Y @ Y.T - alpha * D @ D.T
    _, eigenvectors = eig(mat)
    A = eigenvectors[:, :R]
    U = A @ A.T @ D
    V = (inv(U.T @ U) @ U.T @ Y).T
    return U, V


def update_T(T_prev, U2, U1):
    return T_prev + U2 - U1


def get_score(Y, U, V):
    return np.linalg.norm(Y - U @ V.T, "fro") ** 2


def plot(rank, result_img, primal_img):

error1 and error2 are significantly different because $U_1$ and $U_2$ are different matrices with small scale factor. As far as we approximate $Y_1$ with $U_1V_1^T$ s.t. $U_1 =U_2$ the scale factor of $U_1V_1^T$ is contained in $V_1$ and $U_1$ remains relatively small in terms of norm. So then we compare $U_1$ and $U_2$ the distance error is small because of the small scale factor of $U_1$ and by the same logic $U_2$.

So if we force $U$ to be orthogonal we could resolve that issue.

$$ \begin{align*} \min \quad & \frac{1}{2} ||Y_1 - U_1V_1^T||_F^2 + \frac{1}{2}||Y_2 - U_2V_2^T||_F^2\\ s.t \quad & U_1^TU_1 = I\\ &U_2^TU_2 = I\\ &U_1^TU_2 = I\\ \end{align*} $$
